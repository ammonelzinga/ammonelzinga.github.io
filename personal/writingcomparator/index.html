This website is a free and live site and is an ongoing project to help people study sacred texts. It has features to compare and search for relevant passages. For each sacred text, I had openAI chunk each chapter into relevant groups. Then I embedded those groups. This method of predeterming relevant groups before embedding drastically saved on storage usage. I used Supabase for the data storage. When a user asks a question, the question is embedded by openAI, and the result is stored and compared semantically to all other embedded groups that match the user's filters to return the most related passages of scripture to that question. Similar functionality is used to find similar passages to a particular verse. The site features:

  • Multi pane interface to compare texts side by side
  • Semantic search tool to ask questions to find relevant passages
  • Lexical search tool
  • Summarizes results using AI
  • Built with Supabase, OpenAI API, React, Typescript, and Vercel
